## ⚠️ 앱(AAB)에 아직 안 들어간 웹 변경 — 다음 앱 빌드(versionCode 2)에 반영
> 웹(GitHub Pages)은 push 로 바로 바뀌지만, **앱은 웹을 복사해 동봉**한 거라 자동으로 안 따라온다.
> 아래를 **한꺼번에 모아** 14일 테스트를 채운 뒤 versionCode 2 로 넣는다.

**Play Console 에 게시된 AAB = versionCode 1 = 웹 `v41`**(2026-07-21 게시, 심사 통과). 그 뒤 웹에만 반영한 것:
- **`v42`~`v45` 이 앱 공유하기**(2026-07-22) — 안드로이드=Play·아이폰=웹 두 링크. 공유는 3곳:
  타이틀(출발 버튼 아래), 엔딩(버튼 둘 아래), 지도 상단바(모드 버튼 왼쪽 `🔗 알려주기` 알약).
  `v45` 에서 타이틀·엔딩은 **버튼 대신 작은 민트 글자 링크**(`.share-link`)로 — 공유는 부차적 동작이라
  큰 컬러 버튼과 안 겹치게. 지도만 알약 유지(툴바라 글자는 어색).
  ⚠️ 안드로이드(Play) 공유 링크는 앱이 **공개 출시돼야** 열림(비공개 테스트 중엔 404).

   `버전 수정` 만 열린다. 그 초안에 AAB 를 얹으면 그대로 versionCode 2 가 되니 **따로 지울 것 없다.**

## 참고 (기술 메모)
- 곡 데이터: `index.html` 안 `const SONGS=[...]` 배열. 각 곡 `notes:[['음이름',박],...]`,
  쉼표는 `['R',박]`. `sig:[분자,분모]`, `pickup`(못갖춘마디 박), `perSystem`(한 줄 마디수).
- 6/8은 한 마디 3.0박(8분음표 6개), 4/4는 4.0박. buildMeasures 가 박 합으로 마디를 나눔.
- 폰트 재-subset 스크립트/방법은 세션 스크래치패드에 있었음(원본 구글폰트에서 재생성).
- 악보를 이미지로 확인시켜줄 때: 게임의 drawNote/drawBeams 로직을 그대로 옮긴
  Node 스크립트로 SVG 생성해서 보여줌.
- 점검 도구(`sarangnara-app/`): `inspect-phone.mjs`(폰 웹뷰 상태 한 번 훑기),
  `walk-phone.mjs`(폰에서 타이틀→지도→대화→연주까지 눌러가며 화면마다 점검),
  `shot-play.mjs`(패드·아이폰·S8 크기로 **모든 곡 × 연습/전체 모드** 연주 화면 레이아웃 검사).
  폰에 붙는 법: 디버그 APK 설치 → `adb shell pidof com.sya.sarangnara` →
  `adb forward tcp:9222 localabstract:webview_devtools_remote_<pid>`.
  **화면이 꺼져 있으면 innerWidth 가 0 이라 측정이 다 0 으로 나온다** — 깨워 놓고 잴 것.
